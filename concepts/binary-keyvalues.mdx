## Overview

Source 2 uses two KeyValues binary formats for structured data storage:

- **KV1** (KeyValues1) - Legacy binary format from Source 1
- **KV3** (KeyValues3) - Modern binary format with versioning and compression

These formats are used extensively in DATA blocks and other specialized blocks.

## Binary KV1 Format

### Magic Number

```csharp
// From BinaryKV1.cs:14
public const int MAGIC = 0x564B4256; // "VBKV"
```

KV1 files start with the magic bytes `VBKV` (0x564B4256).

### Structure

```csharp
// From BinaryKV1.cs:9-54
public class BinaryKV1 : Block
{
    public override BlockType Type => BlockType.DATA;
    
    /// <summary>
    /// Gets the deserialized KeyValues data.
    /// </summary>
    public KVObject? KeyValues { get; private set; }

    public override void Read(BinaryReader reader)
    {
        reader.BaseStream.Position = Offset;
        KeyValues = KVSerializer.Create(KVSerializationFormat.KeyValues1Binary)
            .Deserialize(reader.BaseStream);
    }
}
```

### Usage

KV1 is the legacy format, rarely used in modern Source 2 resources:

```csharp
using var resource = new Resource();
resource.Read("legacy_data.vdata_c");

if (resource.DataBlock is BinaryKV1 kv1)
{
    var data = kv1.KeyValues;
    // Access KV1 data
}
```

## Binary KV3 Format

### Magic Numbers

KV3 supports multiple versions:

```csharp
// From BinaryKV3.cs:26-53
public const int MAGIC0 = 0x03564B56; // VKV3 (3 isn't ASCII, it's 0x03)
public const int MAGIC1 = 0x4B563301; // KV3\x01
public const int MAGIC2 = 0x4B563302; // KV3\x02
public const int MAGIC3 = 0x4B563303; // KV3\x03
public const int MAGIC4 = 0x4B563304; // KV3\x04
public const int MAGIC5 = 0x4B563305; // KV3\x05

public static bool IsBinaryKV3(uint magic) => 
    magic is MAGIC0 or MAGIC1 or MAGIC2 or MAGIC3 or MAGIC4 or MAGIC5;
```

### Structure


### Format Identifier

Each KV3 block has a format GUID that identifies its schema:

```csharp
// From BinaryKV3.cs:208
Format = KV3IDLookup.GetByValue(new Guid(reader.ReadBytes(16)));
```

Common format identifiers include:
- Generic data structures
- Material parameters
- Particle definitions
- Animation data

### Compression Methods

KV3 supports multiple compression methods:

<ResponseField name="0" type="Uncompressed">
  No compression applied
</ResponseField>

<ResponseField name="1" type="LZ4">
  LZ4 compression with 16KB frame size (version 2+)
</ResponseField>

<ResponseField name="2" type="ZSTD">
  Zstandard compression (version 2+)
</ResponseField>

```csharp
// From BinaryKV3.cs:210-376
var compressionMethod = reader.ReadUInt32();

if (compressionMethod == 0) // Uncompressed
{
    reader.Read(buffer1Span);
}
else if (compressionMethod == 1) // LZ4
{
    DecompressLZ4(reader, buffer1Span, sizeCompressedBuffer1);
}
else if (compressionMethod == 2) // ZSTD
{
    DecompressZSTD(zstdDecompressor, reader, buffer1Raw.AsSpan(0, outBufferLength), 
                   sizeCompressedBuffer1);
}
```

### Version Differences

#### Version 1
- Basic structure
- Limited metadata
- No compression dictionary support

#### Version 2-4
- Added compression support (LZ4, ZSTD)
- Compression frame size (16KB)
- Binary blob storage
- Block-based compression for large data

#### Version 5 (Latest)
- Dual buffer system (primary + auxiliary)
- Optimized memory layout
- Improved string storage
- Better handling of typed arrays

```csharp
// From BinaryKV3.cs:270-286
if (version >= 5)
{
    sizeUncompressedBuffer1 = reader.ReadInt32();
    sizeCompressedBuffer1 = reader.ReadInt32();
    sizeUncompressedBuffer2 = reader.ReadInt32();
    sizeCompressedBuffer2 = reader.ReadInt32();
    countBytes1_buffer2 = reader.ReadInt32();
    countBytes2_buffer2 = reader.ReadInt32();
    countBytes4_buffer2 = reader.ReadInt32();
    countBytes8_buffer2 = reader.ReadInt32();
    // ...
}
```

## KV3 Flags

KV3 values can have metadata flags:

```csharp
public enum KVFlag
{
    None = 0,
    Resource = 1,        // Resource path
    ResourceName = 2,    // Resource name without extension
    Panorama = 8,        // Panorama-specific
    SoundEvent = 16,     // Sound event name
    SubClass = 32,       // Subclass identifier
}
```

Flags are stored alongside the type information:

```csharp
// From BinaryKV3.cs:700-748
private static (KV3BinaryNodeType Type, KVFlag Flag) ReadType(Context context)
{
    var databyte = context.Types[0];
    context.Types = context.Types[1..];
    var flagInfo = KVFlag.None;

    if (context.Version >= 3)
    {
        if ((databyte & 0x80) > 0)
        {
            databyte &= 0x3F; // Remove the flag bit
            flagInfo = (KVFlag)context.Types[0];
            context.Types = context.Types[1..];
        }
    }
    
    return ((KV3BinaryNodeType)databyte, flagInfo);
}
```

## Reading KV3 Data

### Basic Usage

```csharp
using var resource = new Resource();
resource.Read("particles/example.vpcf_c");

if (resource.DataBlock is BinaryKV3 kv3)
{
    var data = kv3.Data;
    
    // Access properties
    foreach (var prop in data.Properties)
    {
        Console.WriteLine($"{prop.Key}: {prop.Value}");
    }
}
```

### Accessing Nested Data

```csharp
if (resource.DataBlock is BinaryKV3 kv3)
{
    // Get a specific property
    if (kv3.Data.Properties.TryGetValue("m_nMaxParticles", out var maxParticles))
    {
        Console.WriteLine($"Max Particles: {maxParticles.Int64Value}");
    }
    
    // Navigate nested objects
    if (kv3.Data.Properties.TryGetValue("m_Children", out var children))
    {
        if (children.Value is KVObject childrenObj && childrenObj.IsArray)
        {
            foreach (var child in childrenObj.Properties.Values)
            {
                // Process each child
            }
        }
    }
}
```

### Converting to Text Format

```csharp
if (resource.DataBlock is BinaryKV3 kv3)
{
    // Get as KV3File for text output
    var kv3File = kv3.GetKV3File();
    
    // Write to text format
    using var writer = new IndentedTextWriter();
    kv3File.WriteText(writer);
    Console.WriteLine(writer.ToString());
}
```

## Memory Layout

KV3 uses a sophisticated buffer system to minimize memory usage:

```csharp
// From BinaryKV3.cs:77-95
private class Buffers
{
    public ArraySegment<byte> Bytes1;  // 1-byte values
    public ArraySegment<byte> Bytes2;  // 2-byte values (version 4+)
    public ArraySegment<byte> Bytes4;  // 4-byte values
    public ArraySegment<byte> Bytes8;  // 8-byte values
}

private class Context
{
    public int Version;
    public ArraySegment<byte> Types;
    public ArraySegment<byte> ObjectLengths;
    public ArraySegment<byte> BinaryBlobs;
    public ArraySegment<byte> BinaryBlobLengths;
    public string[] Strings;
    public Buffers Buffer;
    public Buffers AuxiliaryBuffer;  // Version 5+
}
```

Data is packed by size for optimal alignment:
1. All 1-byte values together
2. All 2-byte values aligned to 2-byte boundaries
3. All 4-byte values aligned to 4-byte boundaries
4. All 8-byte values aligned to 8-byte boundaries

## Working with Arrays

### Typed Arrays

```csharp
// Typed arrays store the element type once, then all elements
case KV3BinaryNodeType.ARRAY_TYPED:
    var arrayLength = MemoryMarshal.Read<int>(buffer.Bytes4);
    buffer.Bytes4 = buffer.Bytes4[sizeof(int)..];
    
    var (subType, subFlagInfo) = ReadType(context);
    var typedArray = new KVObject(name, isArray: true, capacity: arrayLength);
    
    for (var i = 0; i < arrayLength; i++)
    {
        ReadBinaryValue(context, name, subType, subFlagInfo, typedArray);
    }
```

### Mixed Arrays

```csharp
// Untyped arrays store type information for each element
case KV3BinaryNodeType.ARRAY:
    var arrayLength = MemoryMarshal.Read<int>(buffer.Bytes4);
    buffer.Bytes4 = buffer.Bytes4[sizeof(int)..];
    
    var array = new KVObject(name, isArray: true, capacity: arrayLength);
    
    for (var i = 0; i < arrayLength; i++)
    {
        ParseBinaryKV3(context, array, true);
    }
```

## Binary Blobs

Large binary data is stored separately:

```csharp
// From BinaryKV3.cs:928-945
case KV3BinaryNodeType.BINARY_BLOB:
    var blockLength = MemoryMarshal.Read<int>(context.BinaryBlobLengths);
    context.BinaryBlobLengths = context.BinaryBlobLengths[sizeof(int)..];
    
    byte[] output;
    if (blockLength > 0)
    {
        output = [.. context.BinaryBlobs[..blockLength]];
        context.BinaryBlobs = context.BinaryBlobs[blockLength..];
    }
    else
    {
        output = [];
    }
    
    parent.AddProperty(name, MakeValue(datatype, output, flagInfo));
```
