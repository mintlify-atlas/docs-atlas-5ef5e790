## Overview

Source 2 supports over 80 different resource types, each identified by a file extension. All compiled resources end with `_c` suffix.

## Common Resource Types

### Models and Meshes

<ResponseField name="vmdl_c" type="Model">
  3D model created in the ModelDoc Editor. Contains mesh references, materials, animations, and physics data.
</ResponseField>

<ResponseField name="vmesh_c" type="Mesh">
  Mesh geometry. Legacy format - mesh data is now typically stored in vmdl files.
</ResponseField>

<ResponseField name="vmorf_c" type="Morph">
  Morph set containing blend shapes and flex data for facial animation.
</ResponseField>

### Materials and Textures

<ResponseField name="vmat_c" type="Material">
  Material definition created in the Material Editor. References textures and defines shader parameters.
</ResponseField>

<ResponseField name="vtex_c" type="Texture">
  Compiled texture in various formats (DXT, BC7, etc.). May contain mipmaps and streaming data.
</ResponseField>

<ResponseField name="vcompmat_c" type="CompositeMaterial">
  Composite material kit for CS2 weapon skins.
</ResponseField>

### Animation

<ResponseField name="vanmgrph_c" type="AnimationGraph">
  Animgraph 1 graph for animation behavior and states.
</ResponseField>

<ResponseField name="vnmgraph_c" type="NmGraph">
  Animgraph 2 graph for animation behavior. Graph variations have a `.vnmgraph.+variationname` suffix.
</ResponseField>

<ResponseField name="vnmclip_c" type="NmClip">
  Animgraph 2 animation clip. Can contain events to respond from graphs.
</ResponseField>

<ResponseField name="vnmskel_c" type="NmSkeleton">
  Animgraph 2 skeleton. Used for skeleton(s) and bonemask data.
</ResponseField>

<ResponseField name="vnmikrig_c" type="NmIKRig">
  Animgraph 2 IK rig. Can contain IK rig data.
</ResponseField>

### World and Maps

<ResponseField name="vwrld_c" type="World">
  World root file containing references to world nodes and global settings.
</ResponseField>

<ResponseField name="vwnod_c" type="WorldNode">
  World node referencing map geometry and scene objects.
</ResponseField>

<ResponseField name="vents_c" type="EntityLump">
  Entity lump containing map entities with their properties and connections.
</ResponseField>

<ResponseField name="vmap_c" type="Map">
  Map definition file created in Hammer.
</ResponseField>

<ResponseField name="vvis_c" type="WorldVisibility">
  World visibility data stored in voxel clusters for occlusion culling.
</ResponseField>

### Particles and Effects

<ResponseField name="vpcf_c" type="Particle">
  Particle system created in the Particle Editor.
</ResponseField>

<ResponseField name="vsnap_c" type="ParticleSnapshot">
  Particle snapshot reference.
</ResponseField>

<ResponseField name="vpost_c" type="PostProcessing">
  Post-processing settings created in the Postprocessing Editor.
</ResponseField>

### Audio

<ResponseField name="vsnd_c" type="Sound">
  Sound file or sound container. May contain streaming audio data.
</ResponseField>

<ResponseField name="vsndevts_c" type="SoundEventScript">
  Sound event script defining sound events.
</ResponseField>

<ResponseField name="vsndstck_c" type="SoundStackScript">
  Sound stack script with rules and operators for sound events.
</ResponseField>

<ResponseField name="vmix_c" type="VMix">
  Mix graph for audio mixing created in the VMix Tool.
</ResponseField>

### Panorama UI

<ResponseField name="vxml_c" type="PanoramaLayout">

</ResponseField>

<ResponseField name="vcss_c" type="PanoramaStyle">

</ResponseField>

<ResponseField name="vjs_c" type="PanoramaScript">
  Panorama JavaScript for UI logic.
</ResponseField>

<ResponseField name="vts_c" type="PanoramaTypescript">
  Panorama TypeScript for UI logic.

<ResponseField name="vsvg_c" type="PanoramaVectorGraphic">

</ResponseField>

<ResponseField name="vpdi_c" type="PanoramaDynamicImages">

### Physics

</ResponseField>

### Game Data

## Determining Resource Type

The library determines the resource type using multiple methods:

```csharp
// From Resource.cs:201-288
// 1. By file extension
ResourceType = ResourceTypeExtensions.DetermineByFileExtension(
    Path.GetExtension(FileName.AsSpan())
);

// 2. By compiler identifier in REDI block
foreach (var specialDep in EditInfo.SpecialDependencies)
{
    ResourceType = DetermineResourceTypeByCompilerIdentifier(specialDep);
    if (ResourceType != ResourceType.Unknown) break;
}

// 3. By input dependency extension if only one exists
if (ResourceType == ResourceType.Unknown && EditInfo.InputDependencies.Count == 1)
{
    ResourceType = ResourceTypeExtensions.DetermineByFileExtension(
        Path.GetExtension(EditInfo.InputDependencies[0].ContentRelativeFilename)
    );
}
```

## Compiler Identifiers

Some common compiler identifier mappings:

```csharp
// From Resource.cs:587-643
"CompileAnimgraph" => ResourceType.AnimationGraph
"CompileRenderMesh" => ResourceType.Mesh
"CompileTexture" => ResourceType.Texture
"CompileMaterial" => ResourceType.Material
"CompileModel" => ResourceType.Model
"CompileParticle" => ResourceType.Particle
"CompileWorld" => ResourceType.World
"CompileVPhysXData" => ResourceType.PhysicsCollisionMesh
"CompileSmartProp" => ResourceType.SmartProp
"CompilePanorama" => ResourceType.Panorama (with subtype detection)
```

## Block Construction by Type

Different resource types construct different DATA blocks:

```csharp
// From Resource.cs:535-567
ResourceType switch
{
    ResourceType.AnimationGraph => new AnimGraph(),
    ResourceType.NmClip => new AnimationClip(),
    ResourceType.EntityLump => new EntityLump(),
    ResourceType.Map => new Map(),
    ResourceType.Material => new Material(),
    ResourceType.Mesh => new Mesh(),
    ResourceType.Model => new Model(),
    ResourceType.Morph => new Morph(),
    ResourceType.Panorama => new Panorama(),
    ResourceType.Particle => new ParticleSystem(),
    ResourceType.PhysicsCollisionMesh => new PhysAggregateData(),
    ResourceType.PostProcessing => new PostProcessing(),
    ResourceType.Sound => new Sound(),
    ResourceType.Texture => new Texture(),
    ResourceType.World => new World(),
    ResourceType.WorldNode => new WorldNode(),
    // ... many more
}
```

## Example: Working with Resource Types

```csharp
using ValveResourceFormat;

// Load a resource
using var resource = new Resource();
resource.Read("materials/dev/dev_metal.vmat_c");

// Check resource type
if (resource.ResourceType == ResourceType.Material)
{
    var material = (Material)resource.DataBlock;
    // Work with material data
}

// Access type-specific data
switch (resource.ResourceType)
{
    case ResourceType.Texture:
        var texture = (Texture)resource.DataBlock;
        // Extract texture data
        break;
        
    case ResourceType.Model:
        var model = (Model)resource.DataBlock;
        // Access model meshes
        break;
        
    case ResourceType.Sound:
        var sound = (Sound)resource.DataBlock;
        // Extract audio data
        break;
}
```

## Legacy Types

Some resource types are marked as legacy:

- **vanim_c** (Animation) - Data now stored in vmdl files
- **vagrp_c** (AnimationGroup) - Data now stored in vmdl files  
- **vseq_c** (Sequence) - Data now stored in vmdl files
- **vmesh_c** (Mesh) - Data now stored in vmdl files
- **vpsf_c** (ParticleSnapshotLegacy) - Replaced by vsnap

<Warning>
Legacy resource types may have limited support or may be fully deprecated in newer Source 2 versions.
</Warning>
